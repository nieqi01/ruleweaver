# RuleWeaver 项目开发总结

## 项目概述

**RuleWeaver** 是一款离线优先的 PWA 应用，专为规则怪谈网文作者设计，帮助管理复杂的世界观设定，防止设定崩坏。

## 技术实现要点

### 1. 离线优先架构
- **IndexedDB**: Dexie.js 封装，支持 50MB+ 数据
- **Service Worker**: 缓存核心资源，离线可用
- **PWA**: 可添加到主屏幕，类似原生应用体验

### 2. 数据模型
- 副本（Instance）作为顶层容器
- 实体（Entity）统一存储角色/道具/规则
- 关系（Relation）独立表，支持多对多
- 地图标记（MapMarker）存储归一化坐标

### 3. 移动端适配
- CSS Media Query 断点：768px
- 底部导航栏替代侧边栏
- Cytoscape 性能不足时自动降级列表视图
- 触摸优化：大按钮、底部滑出模态框

### 4. AI 集成
- 前端直连 Kimi API，用户自备 Key
- Key 存储在 localStorage，不上传服务器
- Prompt 模板针对规则怪谈场景优化

### 5. 数据安全
- 导出支持 AES-256 加密（crypto-js）
- 数据完全本地存储
- 自动备份到下载文件夹

## 使用流程

```
1. 创建副本（如：青山中学）
   ↓
2. 添加实体（角色/道具/规则/地图）
   ↓
3. 建立关系（冲突/伏笔/因果/持有）
   ↓
4. 上传地图并标记实体位置
   ↓
5. 写作章节，使用标记语法 {{角色:张三}}
   ↓
6. 章节体检，检查伏笔回收和状态矛盾
   ↓
7. AI 评估规则合理性
   ↓
8. 导出加密设定包分享给协作者
```

## 文本标记语法

| 语法 | 含义 | 示例 |
|-----|------|------|
| `{{类型:名称}}` | 引用实体 | `{{角色:张三}}` |
| `[[道具名]]` | 道具简写 | `[[染血的校规]]` |
| `<<伏笔名:埋>>` | 埋下伏笔 | `<<照片里的手:埋>>` |
| `<<伏笔名:收>>` | 回收伏笔 | `<<照片里的手:收>>` |
